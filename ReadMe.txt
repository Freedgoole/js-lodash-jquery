Добрый день! 

Demo: http://news-api.surge.sh/

Проект разбит по функционалу


1. LocalStorage 
    Добавление данных в хранилище
    Получение данных из хранилище
    Очистка 
    return методов в главный модуль app

2. NewApi 
    загрузка API ресурса 
    return полученный масив новостей в главный модуль app

3.Ui
    Статически определены контейнеры для вывода данных (возможно вынести, типа init)
    Валидация даты, так как в изначальном массиве были буквы RegEx вынесен в консруктор (так как функция не большая, не выносил в отдельную функцию)
    Ограничение на колличество символов title при hover на источниках (так как функция не большая, не выносил в отдельную функцию)
    Валация поля author (исходный массив не всегда дает корректные данные)
    Добавлена картинка по default на новостной блок (!)
    Вывод сообщения пользователю если произошел сбой при загрузке с  Api
    Модуль валидации можно вынести отдельно с массивом параметров
    config

4.app (head)
    инициализация
    Проверка на наличие хранилища в браузере и на его содержимое
    Сценарий загрузки вывода сообщений

5.index.js 
    Связь компонентов ..
